## 27.3 length 프로퍼티와 희소 배열
- length 프로퍼티의 값은 배열에 요소 추가, 삭제 시 자동 갱신
```js
const arr = [1, 2, 3];
console.log(arr.length); // 3

arr.push(4);
console.log(arr.length); // 4

arr.pop();
console.log(arr.length); // 3
```

```js
// 희소 배열
const sparse = [, 2, , 4];
console.log(sparse.length); // 4
console.log(sparse); // [empty, 2, empty, 4]
// → 인덱스가 0, 2인 요소가 존재하지 않음
// → 희소
```
- 위의 배열에서 인덱스가 0, 2인 요소가 존재하지 않음
- 희소 배열의 length는 희소 배열의 실제 요소 개수보다 *언제나 크다*.
- 배열에는 같은 타입의 요소를 연속적으로 위치시키는 것이 최선!
  → 희소 배열을 생성하지 않도록 주의!

## 27.4 배열 생성
  ### 27.4.1 배열 리터럴
  - 가장 일반적이고 간편한 배열 생성 방식
  ```js
  const arr = [1, 2, 3];
  ```

  ### 27.4.2 Array 생성자 함수
  - 전달된 인수가 1개 & 숫자 → length 프로퍼티 값이 인수인 배열 생성
  - 이때 생성된 배열 = 희소 배열 → length 프로퍼티 값은 10, but 요소가 존재하지 X. 
  ```js
  const arr = new Array(10);

  console.log(arr); // [empty * 10]
  console.log(arr.length); // 10

  console.log(Object.getOwnPropertyDescriptors(arr));
  // { length: {value: 10, writable: true, enumerable: false, configurable: false} }
  ``` 

  ### 27.4.3 Array.of
  - 전달된 인수가 1개 & 숫자 → 인수를 요소로 갖는 배열 생성
  ```js
  Array.of(1); // [1]
  Array.of(1, 2, 3); // [1, 2, 3] 
  Array.of('string'); // ['string']
  ```

  ### 27.4.4 Array.from
  - *유사 배열 객체* or *이터러블 객체*를 인수로 전달받아 배열로 변환
  ```js
  // 유사 배열 객체
  Array.from({ length: 2, 0: 'a', 1: 'b' }); // ['a', 'b']

  // 이터러블 객체 (문자열)
  Array.from('Hello'); // ['H', 'e', 'l', 'l', 'o']

  // length만 있으면 요소는 undefined
  Array.from({ length: 3 }); // [undefined, undefined, undefined]

  // 두 번째 인수로 전달한 콜백 함수의 반환값으로 배열 구성
  Array.from({ length: 3 }, (_, i) -> i); // [0, 1, 2]
  ```

## 27.5 배열 요소의 참조
```js
const arr = [1, 2];

// 인덱스가 0인 요소를 참조
console.log(arr[0]); // 1
```

## 27.6 배열 요소의 추가와 갱신
```js
const arr = [1, 2];

// 배열 요소 추가
arr[2] = 100;

console.log(arr); // [1, 2, 100]
console.log(arr.length); // 3

// 배열 요소 갱신
arr[1] = 10;

console.log(arr); // [1, 10, 100]
```

## 27.7 배열 요소의 삭제
```js
const arr = [1, 2, 3];

delete arr[1];
console.log(arr); // [1, empty, 3]
console.log(arr.length); // 3

arr.splice(1, 1);
console.log(arr); // [1, 3]
console.log(arr.length); // 2
```

## 27.8 배열 메서드
- 결과물을 반환하는 두 가지 방법
  1. 원본 배열을 직접 변경하는 메서드 → push()
  2. 새로운 배열을 생성하여 반환하는 메서드 → concat()

  ### 27.8.1 Array.isArray
  - Array 생성자 함수의 정적 메서드
  ```js
  // true
  Array.isArray([]);
  Array.isArray([1, 2]);
  Array.isArray(new Array());

  // false
  Array.isArray();
  Array.isArray({});
  Array.isArray(true);
  ```

  ### 27.8.2 Array.prototype.indexOf
  - 원본 배열에서 인수로 전달된 요소를 검색하여 인덱스를 반환
  - 배열에 특정 요소의 존재 여부 확인에 좋음
  ```js
  const arr = [1, 2, 2, 3];

  arr.indexOf(2); // 1
  arr.indexOf(4); // -1
  // 두 번째 인수는 검색을 시작할 인덱스
  arr.indexOf(2, 2); // 2
  ```

  ### 27.8.3 Array.prototype.push
  ```js
  const arr = [1, 2];

  let result = arr.push(3, 4);
  console.log(result); // 4
  console.log(arr); // [1, 2, 3, 4]
  ```

  ### 27.8.4 Array.prototype.pop
  ```js
  const arr = [1, 2];

  let result = arr.pop();
  console.log(result); // 2
  console.log(arr); // [1]
  ```

  ### 27.8.5 Array.prototype.unshift
  - 인수로 전달받은 모든 값을 원본 배열의 선두에 요소 추가
  ```js
  const arr = [1, 2];

  let result = arr.unshift(3, 4);
  console.log(result); // 4
  console.log(arr); // [3, 4, 1, 2]
  ```

  ### 27.8.6 Array.prototype.shift
  - 첫 번째 요소 제거 → 제거 요소 반환
  ```js
  const arr = [1, 2];

  let result = arr.shift();
  console.log(result); // 1
  console.log(arr); // [2]
  ```

  ### 27.8.7 Array.prototype.concat
  - 마지막 요소로 추가한 새로운 배열 반환
  - 원본 배열은 변경되지 않는다.
  ```js
  const arr1 = [1, 2];
  const arr2 = [3, 4];

  let result = arr1.concat(arr2);
  console.log(result); // [1, 2, 3, 4]
  
  result = arr1.concat(3);
  console.log(result); // [1, 2, 3]

  let result = arr1.concat(arr2, 5);
  console.log(result); // [1, 2, 3, 4, 5]

  console.log(arr1); // [1, 2]
  ```

  ### 27.8.8 Array.prototype.splice
  ```js
  const arr = [1, 2];

  let result = arr.splice(1, 2, 20, 30);
  console.log(result); // [2, 3]
  console.log(arr); // [1, 20, 30, 4]
  ```

  ### 27.8.9 Array.prototype.slice
  - 원본 배열의 복사본을 생성 → 생성된 복사본은 `얕은 복사`를 통해 생성
  ```js
  const arr = [1, 2, 3];

  arr.slice(0, 1); // [1]
  arr.slice(1, 2); // [2]

  // 원본은 변경되지 않는다.
  console.log(arr); // [1, 2, 3]
  ```

  ### 27.8.10 Array.prototype.join
  - 구분자로 연결한 문자열을 반환, 생략 가능
  ```js
  const arr = [1, 2, 3, 4];

  arr.join(); // '1,2,3,4'
  arr.join(''); // '1234'
  arr.join(':'); // '1:2:3:4'
  ```

  ### 27.8.11 Array.prototype.reverse
  ```js
  conat arr = [1, 2, 3];
  const result = arr.reverse();

  // 원본 배열을 직졉 변경
  console.log(arr); // [3, 2, 1]
  // 반환값은 변경한 배열
  console.log(result); // [3, 2, 1]
  ```

  ### 27.8.12 Array.prototype.fill
  - 인수로 전달받은 값을 배열의 처음부터 끝까지 요소로 채운다.
  - 원본 배열이 변경
  ```js
  const arr = [1, 2, 3];

  // 배열 전체를 인수로 채움
  arr.fill(0);
  // 원본 배열의 변경
  console.log(arr); // [0, 0, 0]
  ```

  ### 27.8.13 Array.prototype.includes
  - 배열 내에 특정 요소가 포함되어 있는지 확인 (true / false)
  ```js
  const arr = [1, 2, 3];
  
  arr.includes(2); // true

  arr.includes(100); // false

  [NaN].indexOf(NaN) !== -1 // false
  [NaN].includes(NaN) // true
  ```

  ### 27.8.14 Array.prototype.flat
  ```js
  [1, [2, 3, 4, 5]].flat(); // [1, 2, 3, 4, 5]

  // 인수로 infinity를 전달하면 모두 적용
  [1, [2, [3, [4]]]].flat(infinity); // [1, 2, 3, 4]
  ```

## 27.9 배열 고차 함수

  ### 27.9.1 Array.prototype.sort
  - 기본적으로 오름차순으로 정렬
  - 원본 배열을 직접 변경
  ```js
  const fruits = ['Banana', 'Orange', 'Apple'];

  fruits.sort();

  console.log(fruits); // ['Apple', 'Banana', 'Orange']

  // 내림차순을 원하면 sort() 먼저 실행 후 reverse 메서드를 사용
  fruits.reverse();

  console.log(fruits); // ['Orange', 'Banana', 'Apple']
  ```

  ### 27.9.2 Array.prototype.forEach
  ```js
  const numbers = [1, 2, 3];
  const pows = [];

  // for 문으로 배열 순회
  for (let i = 0; i < numbers.length; i++) {
    pows.push(numbers[i] ** 2);
  }

  // for문 대신 사용하는 forEach 메서드
  // numbers 배열의 모든 요소 순회 → 콜백 함수 반복 호출
  numbers.forEach(item => pows.push(item ** 2));

  console.log(pows); // [1, 4, 9]
  ```

  ### 27.9.3 Array.prototype.map
  ```js
  const numbers = [1, 4, 9];

  const roots = numbers.map(item => Math.sqrt(item));
  // 위의 코드는 다음과 같다.
  const roots = numbers.map(Math.sqrt());

  // 새로운 배열을 반환
  console.log(roots); // [1, 2, 3]
  // 원본 배열 변겅하지 않음
  console.log(numbers); // [1, 4, 9]
  ```

  ### 27.9.4 Array.prototype.filter
  - 배열의 모든 요소를 순회하면서 콜백 함수 반복 호출
  - 콜백 함수 반환값이 true인 요소로만 구성된 새로운 배열을 반환
  - 새로운 배열의 length 프로퍼티 값은 filter 메서드를 호출한 배열의 length 프로퍼티 값과 같거나 작다.
  ```js
  const numbers = [1, 2, 3, 4, 5];

  // 
  const odds = numbers.filter(item => item % 2);
  console.log(odds);
  ```

  ### 27.9.5 Array.prototype.reduce
  - 하나의 결과값을 반환
  - 언제나 초기값을 전달하는 것이 안전 → 빈 배열은 에러 발생
  - 
  ```js
  const sum = [1, 2, 3, 4].reduce((accumulator, currentValue, index, array) => accumulator + currentValue, 0);

  console.log(sum); // 10 → 하나의 결과값
  ```

  ### 27.9.6 Array.prototype.some
  - 배열의 요소 중 조건에 해당하는 것이 1개 이상 존재하는지 확인
  ```js
  [5, 10, 15].some(item => item > 10); // true

  ['Apple', 'Banana', 'Mango'].some(item => item === 'Banana'); // true

  // 빈 배열은 언제나 false 반환
  [].some(item => item > 3); // false
  ```

  ### 27.9.7 Array.prototype.every
  - 배열의 모든 요소가 조건보다 큰지 확인
  ```js
  [5, 10, 15].every(item => item > 3); // true

  [5, 10, 15].every(item => item > 10); // false

  // 빈 배열은 언제나 true 반환
  [].every(item => item > 3); // true
  ```

  ### 27.9.8 Array.prototype.find
  ```js
  const users = [
    { id: 1, name: 'Lee' },
    { id: 2, name: 'Kim' },
    { id: 3, name: 'Choi' },
    { id: 4, name: 'Park' },
  ];

  // id가 2인 첫 번째 요소를 반환
  users.find(user => user.id === 2); //  { id: 2, name: 'Kim' }

  // filter 메서드는 배열을 반환
  [1, 2, 2, 3].filter(item => item === 2); // [2, 2]
  // find 메서드는 요소를 반환
  [1, 2, 2, 3].find(item => item === 2); // 2
  ```

  ### 27.9.9 Array.prototype.findIndex
  ```js
  const users = [
    { id: 1, name: 'Lee' },
    { id: 2, name: 'Kim' },
    { id: 3, name: 'Choi' },
    { id: 4, name: 'Park' },
  ];

  users.findIndex(user => user.id === 2); // 1

  // 콜백 함수 추상화
  function predicate(key, value) {
    // key, value를 기억하는 클로저 반환
    return item => item[key] === value;
  }

  // id가 2인 요소의 인덱스 구하기
  users.findIndex(predicate('id', 2)); // 1
  ```

  ### 27.9.10 Array.prototype.flatMap
  - map을 통해 생성된 새로운 배열을 평탄화
  ```js
  conat arr = ['hello', 'world'];

  // map과 flat를 순차적으로 실행
  arr.map(x => x.split('')).flat(); // ['h','e','l','l','o','w','o','r','l','d']

  // map을 통해 생성된 새로운 배열을 평탄화
  arr.flatMap(x => x.split('')); // ['h','e','l','l','o','w','o','r','l','d']

  // flatMap은 1단계만 평탄화
  arr.flatMap((str, index) => [index, [str, str.length]]);
  // [[0, ['hello', 5]], [1, ['world', 5]]] => [0, ['hello', 5], 1, ['world', 5]]

  // 평탄화 깊이 지정 시, map과 flat을 각각 호출
  arr.map((str, index) => [index, [str, str.length]]).flat(2);
  // [[0, ['hello', 5]], [1, ['world', 5]]] => [0, ['hello', 5], 1, ['world', 5]]
  ```
